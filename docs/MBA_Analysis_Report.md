# Market Basket Analysis ML Program Report

## 1. Business Context and System Goal

This project implements a market basket analysis system for a school supplies store. The system is designed for two primary users:

- Store owner or admin: monitors trends, bundles, promos, and strategy-level insights.
- Cashier: receives real-time cross-sell and frequently bought together suggestions during checkout.

The system improves decisions in product placement, promo planning, homepage ranking, and add-on recommendations. Business value is created through better basket composition, increased average order value, and improved promo targeting.

## 2. Dataset A and Dataset B Description

The system uses at least two transaction datasets that satisfy assignment requirements.

- Dataset A: baseline behavior profile.
- Dataset B: alternate behavior profile for cross-dataset comparison.

Each dataset contains transaction-level rows in long format with fields:

- TransactionID

The backend performs normalization to reduce naming inconsistencies, and diagnostics are exposed for raw unique items, normalized unique items, and unmapped terms.

## 4. Algorithm Choice and Comparison to Apriori

The system uses FP-Growth as the primary mining engine and includes an Apriori comparison endpoint for benchmarking.

Why FP-Growth is selected:

- Better fit for denser baskets.
- Avoids repeated candidate generation overhead of Apriori.
- Scales better in memory and runtime for larger transaction sets.

Comparison dimensions:

- Frequent itemset count
- Rules generated
- Average lift
- Runtime seconds

## 5. Intelligent Mechanisms and Automation

Implemented intelligent mechanisms:

1. Adaptive thresholding: minsup and minconf are adjusted based on data behavior and drift context.
2. Drift detection: rule-set shift tracked using Jaccard similarity.
3. Stability assessment: top-rule survival checked across iterations.
4. Composite rule scoring: support, confidence, and lift are combined into a practical ranking signal.

The pipeline runs without manual threshold retuning and updates outputs automatically when data changes.

## 8. Limitations and Future Improvements

Current limitations:

- Rule quality can vary with sparse baskets.
- Promo simulation does not yet model inventory constraints.
- Drift simulation is synthetic and can be expanded with time-aware transactions.

Future improvements:

- What-if threshold simulation UI
- Affinity heatmap for shelf placement
- Funnel conversion analytics for recommendation effectiveness
